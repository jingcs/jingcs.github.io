运行docker
唯一不同的是运行镜像时时sudo docker run -d -p 5000:80 xxflask
这里前的5000是服务器端口，后面80是之前app.py里Dockfile里写的端口（本来也该写5000懒得改文件了
意思是把docker在80端口的软件映射到服务器的5000

nginx创建映射

nginx的作用：不用nginx也可以直接让用户访问80端口
但这样会有安全问题&一个端口只能一个应用
而大部分的web都是80端口，所以nginx用来转发。
即用户访问80端口，nginx监听这个端口，把他转发给服务器内的端口。
